## [x] 【UI渲染】應正確渲染儀表板的基礎佈局
**範例輸入**：設定 AuthContext 回傳預設 user (`username: 'TestUser', role: 'user'`)
**期待輸出**：頁面上應包含標題「儀表板」、使用者名稱與頭像、一般用戶標籤、以及登出按鈕。此時不應出現「管理後台」的連結。

---

## [x] 【權限控制】當使用者角色為 admin 時，應顯示管理後台連結與管理員標籤
**範例輸入**：設定 AuthContext 回傳預設 user (`username: 'AdminUser', role: 'admin'`)
**期待輸出**：頁面上方的 nav 中應顯示「🛠️ 管理後台」連結，且角色標籤應顯示「管理員」。

---

## [x] 【API互動 - 狀態】畫面初始時應顯示載入中狀態
**範例輸入**：商品 API (getProducts) 尚未解析完成
**期待輸出**：顯示「載入商品中...」的 spinner 區域，且不顯示商品列表與錯誤訊息。

---

## [x] 【API互動 - 成功】成功取得商品資料時應渲染商品列表
**範例輸入**：商品 API 回傳 2 筆商品資料的 Mock array
**期待輸出**：不再顯示載入中，並且畫面上正確渲染這 2 筆商品名稱與價格等資訊。

---

## [x] 【API互動 - 失敗】當 API 回傳一般錯誤時應顯示錯誤訊息
**範例輸入**：商品 API 以 500 狀態碼駁回，並帶有錯誤訊息「伺服器異常」
**期待輸出**：不再顯示載入中，且畫面上應顯示錯誤警告區塊並呈現「伺服器異常」或預設「無法載入商品資料」。

---

## [x] 【API互動 - 權限逾時】當 API 回傳 401 錯誤時不應設定錯誤訊息
**範例輸入**：商品 API 以 401 狀態碼駁回
**期待輸出**：不應顯示錯誤警告區塊 (由 interceptor 接管處理)，且結束載入中狀態。

---

## [x] 【API與狀態互動】點擊登出按鈕應呼叫 logout 並導向至 Login
**範例輸入**：點擊「登出」按鈕
**期待輸出**：應呼叫 `logout` 函式，且路由導向至 `/login` (`replace: true`)。
